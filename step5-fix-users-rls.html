Step 5: 保護者が生徒名を参照できるよう RLS を修正

問題の原因

users テーブルの RLS（Row Level Security）が有効になっており、現状は 自分のレコードしか参照できない ポリシーになっています。

そのため、保護者ログイン時に紐づいた生徒の名前を取得しようとしても null が返り、「—」と表示されてしまいます。

Supabase で実行する SQL

1
supabase.com/dashboard を開く
あいおい塾プロジェクト → 左メニュー「SQL Editor」をクリック
2
以下の SQL を貼り付けて「Run」を押す 
-- ① 既存の users SELECT ポリシーを確認・削除（存在する場合）
-- （既にポリシーがある場合は名前を確認して削除してください）

-- ② users テーブルの新しい RLS ポリシーを追加
-- 自分自身 OR 管理者 OR 紐づいた保護者・生徒が参照可能

CREATE POLICY "users_select_policy"
  ON public.users
  FOR SELECT
  USING (
    -- 自分自身
    id = auth.uid()
    -- 管理者
    OR public.my_role() = 'admin'
    -- 保護者として紐づいた生徒のプロフィールを参照可
    OR EXISTS (
      SELECT 1 FROM public.parent_student_links
      WHERE parent_user_id = auth.uid()
        AND student_user_id = public.users.id
    )
    -- 生徒として紐づいた保護者のプロフィールを参照可
    OR EXISTS (
      SELECT 1 FROM public.parent_student_links
      WHERE student_user_id = auth.uid()
        AND parent_user_id = public.users.id
    )
  );
⚠️ 注意： 既に users テーブルに SELECT ポリシーが存在する場合は、先に削除してから実行してください。
確認方法: Supabase ダッシュボード → Table Editor → users → Policies タブ

既存ポリシーがある場合の対処

既存ポリシーが「自分のみ参照可」のような名前で存在する場合、まず削除します：

-- 既存ポリシーの削除（名前は実際のものに合わせてください）
DROP POLICY IF EXISTS "Users can view own profile." ON public.users;
DROP POLICY IF EXISTS "ユーザーは自分のプロフィールを参照可" ON public.users;

-- その後、上記の新しいポリシーを実行

確認方法

1
SQL 実行後、保護者ダッシュボード を開いてリロード
2
「お子様の状況」に生徒の名前（例: 山田 太郎）が表示されていれば成功
✅ 名前が表示されれば完了です！
